To configure a Drupal backend with the provided structure, you will want to create custom content types and ensure they are exposed via the JSON:API. Below is a step-by-step guide to achieve this.

### Step 1: Define Content Types

#### Content Type: Header

1. **Name**: Header
2. **Fields**:
   - **Logo**: (Text Field) - Machine Name: `field_logo`
   - **Navigation**: (List (text)) - Machine Name: `field_navigation`

#### Content Type: Main

1. **Name**: Main
2. **Fields**:
   - **Banner Title**: (Text Field) - Machine Name: `field_banner_title`
   - **Banner Subtitle**: (Text Field) - Machine Name: `field_banner_subtitle`
   - **Banner Buttons**: (Entity Reference Revisions to Banner Button Content Type) - Machine Name: `field_banner_buttons`

#### Content Type: Banner Button

1. **Name**: Banner Button
2. **Fields**:
   - **Text**: (Text Field) - Machine Name: `field_text`
   - **Action**: (Text Field) - Machine Name: `field_action`

### Step 2: Configure JSON:API

Ensure that JSON:API is enabled and correctly configured to expose the above content types. By default, JSON:API exposes all content types, but you can verify by navigating to `/admin/config/services/jsonapi`.

### Step 3: Example JSON:API Configuration

When you create instances of these content types, they will be available via JSON:API. Assuming the JSON:API module is enabled, you can fetch these resources with the following expected endpoints:

- **Header**: `/jsonapi/node/header`
- **Main**: `/jsonapi/node/main`
- **Banner Button**: `/jsonapi/node/banner_button`

Here is a sample configuration using the given JSON data:

1. **Create a Header Node**:

### Step 5: Test JSON:API

Once you have created the nodes, test the JSON:API endpoints using a tool like Postman or a simple cURL request to ensure you get the proper JSON responses for each content type.

This setup will help you maintain structured content for integration with front-end applications, offering flexibility in content management and consumption through the JSON:API.
