To generate a Drupal backend configuration for the given data, you need to create content types and potentially custom blocks to store and manage the configuration data for the header and hero section. Furthermore, if you want this to be exposed via the JSON:API, you need to ensure those endpoints are enabled and accessible.

### Step 1: Create Content Types

1. **Content Type: Global Header**

   - **Fields:**
     - `field_logo`: Text field for storing the logo text.
     - `field_navigation`: Entity reference (taxonomy term reference) or a simple list for storing navigation items. For simplicity, we might use a List (text) field.

2. **Content Type: Hero Section**

   - **Fields:**
     - `field_background_image`: Image field for uploading the background image.
     - `field_text_line1`: Text field for the first line of text.
     - `field_text_line2`: Text field for the second line of text.
     - `field_buttons`: Entity reference for a separate content type or custom block, or a complex custom field type combining a Link and text.

3. **Content Type: Button**

   - **Fields:**
     - `field_button_text`: Text field for the button label.
     - `field_button_link`: Link field for the button URL.

### Step 2: Add JSON:API Configuration

Ensure the JSON:API module is enabled for Drupal and that permissions are set so that the necessary endpoints are accessible.

**Configuring JSON:API:**

- JSON:API comes with default configurations that automatically expose content types. By default, when you define a new content type, it will be available under the JSON:API prefix. 
- For example, the `Global Header` content type would be exposed under `/jsonapi/node/global_header` and the `Hero Section` will be under `/jsonapi/node/hero_section`.

### JSON:API Example Usage

This can be accessed via HTTP GET requests to the appropriate endpoints once content is created. Ensure data is populated by creating nodes for these content types.

### Summary

Here’s how the content types might look semantically:

**Content Type: Global Header**

- `Title`: "Main Navigation Header" (Example usage)
- `field_logo`: "ONEX"
- `field_navigation` (text list): ["Home", "About", "Services", "Team", "Portfolio", "Testimonials", "Blog", "Price Table", "Contact"]

**Content Type: Hero Section**

- `Title`: "Main Hero Section" (Example usage)
- `field_background_image`: Upload an image of dinnerware with wooden utensils.
- `field_text_line1`: "We Believe in Quality"
- `field_text_line2`: "Onex Creative Theme"
- `field_buttons`: Create multiple `Button` content types:
  - `field_button_text`: "Discover Now"
  - `field_button_link`: "#discover"
  - `field_button_text`: "Purchase Onex"
  - `field_button_link`: "#purchase"

This structure will allow Drupal to manage the specified data dynamically and expose it over JSON:API for integration with front-end applications or services.
